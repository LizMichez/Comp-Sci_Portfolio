Javascript - Bug Fixing

2018-10-26

Javascript is a powerfull programming language, but with great power comes great responsibility
so of course there are a lot of ways to mess things up. These mistakes are called errors, and in this assignment
I will be answering questions relating to and explaining these errors.

Define each of the following types of coding errors.

Syntax Error
Basically a grammatical error in code, a syntax error occurs when you forget or put a grammatical symbol in the wrong place. Examples of these symbols would be brackets, braces, semicolons or any other grammatical symbol.
Runtime Error
A runtime error is an error that occurs when a non-syntax error was made in the code that prevents it from running or causes the program to crash. Examples of this error would include calling undefined variables or having a flaw in a program’s math.
Logic Error
A logic error is when all the code is written correctly so it will run, but in a way that will not produce the desired results. In other words, it’s when the programmer makes a mistake with how their code is structured or the order that it’s in.

Explain how you would find each type of error
(debug) in a program.

Debugging syntax errors:
  • Check all closing brackets and braces
  • Check that all semicolons are in the right place
  • Make sure all conditions are correctly formatted
  • Make sure there are quotation marks
  • Comment out code line by line to find error
Debugging runtime errors:
  • Check that all variables are declared
  • Make sure all math is logical
  • Check that all functions are separate
  • Comment out code section by section to find error
Debugging logic errors:
  • Check each function for overlapping or unmentioned conditions
  • Check that everything is in the correct order
  • Draw a flow chart for whichever function is problematic
  • Ask a peer to look over my code
